到目前为止，我们使用的都是属于基本数据类型（整型和实型）的数据，除了基本数据类型C语言还提供了构造数据类型，有数组类型，结构体类型，和共用体类型。构造类型的数据是由基本类型的数据按一定规则组合而成的。

一维数组

一维数组的定义

说明：
（1）数组名的命名规则和变量名相同，遵守标识符命名规则；
（2）在定义数组时必须指定数组中元素的个数即数组的长度，方括号中的表达式就表示这个值；
（3）访问数组元素时，下标是从0开始的；
（4）常量表达式中可以包括常量和符号常量，不能包括变量，也就是说C语言不允许对数组的大小作动态定义；

一维数组的初始化

（1）可以在定义数组时集体赋值；
（2）可以在定义数组之后再逐个赋值；

二维数组

二维数组的定义

说明：其实二维数组也是一维数组，它所有的元素本身就和一维数组一样是一维线性排列的，是连续的，并不存在实际的行和列。所以一定要小心了，因为很多书上都把二维数组画成了行和列的形式，这样比喻是没问题的，但我们一定要注意实际上二维数组并不是按行和列存储的，它就是和一维数组一样是线性连续存储的。

多维数组

像二维数组一样，一个数组的元素可以是另外一个数组，这样就构成了多维数组。

字符数组

字符型数据是以字符的 ASCII 代码存储在存储单元中的，一般占一个字节。由于 ASCII 代码也属于整数形式，因此在 C99 标准中，把字符类型归纳为整型类型中的一种。C 语言中没有字符串类型，字符串是存放在字符型数组中的。

字符数组的定义、初始化及数组元素的引用

用“初始化列表”对字符数组初始化，主要有以下几种情况：
（1）若定义字符数组时不进行初始化，则数组中各元素的值是不可预料的；
（2）若初值个数大于数组长度，则会出现语法错误；
（3）若提供的初值个数小于数组长度，则只将这些字符赋给数组中前面那些元素，其余自动定为’\0’；
（4）若初值个数与预定的数组长度相同，在定义时可以省略数组长度，系统会自动根据初值个数确定数组长度；

字符串和字符串结束标志

在字符数组中，若遇到’\0’，则认为数组中有一个字符串，并表示字符串结束，把它前面的字符组成一个字符串。C 在用字符数组存储字符串常量时会自动加一个’\0’作为结束符，’\0’占一个字节的内存，计入字符数组的长度，但不计入字符串的实际长度。’\0’代表 ASCII 码为 0 的字符，是一个“空字符”，即不显示。

字符串处理函数

库函数并非 C 语言本身的组成部分，而是 C 编译系统为方便用户使用而提供的公共函数。

中的函数
| 一般形式 | 函数功能 |
| ———- | ——————————— |
| puts(字符数组) | 输出一个字符串到终端 |
| gets(字符数组) | 从终端输入一个字符串到字符数组,得到一个函数值(字符数组的起始地址 |

中的函数
| 一般形式 | 函数功能 |
| ——————— | —————————————- |
| strcat(字符数组 1,字符数组 2) | 把数组 2 中的字符串连接到数组 1 中的字符串后面,结果放在数组 1 中,连接时将字符串 1 后的’\0’取消,只在新串最后保留’\0’,函数调用后得到一个函数值(字符数组 1 的起始地址) |
| strcpy(字符数组 1,字符串 2) | 将字符串 2 复制到字符数组 1 中,复制后数组 1 中第一个’\0’后的字符保持原来不变 |
| strcmp(字符串 1,字符串 2) | 比较字符串 1 和字符串 2,按 ASCII 码自左至右逐个字符相比,直到出现不同的字符或遇’\0’为止,比较后得到一个返回值(0,正整数,或负整数) |
| strlen(字符数组) | 返回字符串的实际长度(不包括’ \0’) |
| strlwr(函数) | 将字符串中大写字母转换成小写字母 |
| strupr(字符串) | 将字符串中小写字母转换成大写字母 |
