标准C06-深入理解函数

函数是C语言中模块化程序设计的最小单位，是构成程序的基本模块；

程序的执行从main()的入口开始，到main()的出口结束；

从使用者的角度将函数分为标准库函数和自定义函数两种；

使用ANSI C的库函数必须在程序的开头把函数所在的头文件包含进来；

函数在使用之前必须先声明；

函数名表示符的命名规则与变量的命名规则相同；

函数体内部定义的变量只能在该函数内部访问，称为内部变量，函数头部参数表里的变量称为形式参数，也是内部变量，即只能在函数体内访问；

主调函数把实参的值赋值给被调函数的形参的过程称为参数传递；

函数可以有多个return语句，但返回值只能有一个，类型是除数组以外任何类型；

当函数返回值类型定义为void类型时，表示函数没有返回值，函数可以没有return语句，也可以有”return;”；

print函数是有返回值的，返回值就是print函数输出的字符个数；

在数学中写一个函数有时候可以省略括号，而C语言要求一定要加上括号，例如log(1.0)。在C语言的术语中，1.0是参数，log是函数，log(1.0)是函数调用。

printf也是一个函数，上例中的printf(“sin(pi/2)=%f\nln1=%f\n”, sin(pi/2), log(1.0)) 是带三个参数的函数调用，但是我们并不关心printf函数的返回值（事实上它也是有返回值的，表示实际打印的字符数），而是为了利用它所产生的副作用（Side Effect）－打印。改变计算机存储单元里的数据或者做输入输出操作都算Side Effect。

自定义函数

我们不仅可以调用C标准库提供的函数，也可以定义自己的函数，事实上我们已经这么做了：我们定义了main 函数。main 函数的特殊之处在于执行程序时它自动被操作系统调用，操作系统就认准了main 这个名字。其实操作系统在调用main 函数时是有传参数的，main 函数最标准的形式应该是int main(int argc, char *argv[])，另外C标准也允许int main(void) 这种写法，除了这两种形式之外，定义main 函数的其它写法都是错误的或不可移植的。

main函数是一个自定义函数，除此之外我们还可以再自定义函数。

函数的声明定义和调用

函数原型

函数原型由三部分组成：函数名称，形参列表，返回值类型。

函数声明

在函数原型后直接加上一个；结束，而不写函数体，这称为函数声明。

函数定义

在函数原型后带上函数体，这称为函数定义。

编译器只有见到函数定义才会生成指令，而指令在程序运行时当然也要占存储空间。那么没有函数体的函数声明有什么用呢？它为编译器提供了有用的信息，编译器在翻译代码的过程中，只有见到函数原型(不管带不带函数体)之后才知道这个函数的名字、参数类型和返回值，这样碰到函数调用时才知道怎么生成相应的指令，所以函数原型必须出现在函数调用之前，这也是遵循“先声明后使用”的原则。

C语言的函数定义是互相平行，独立的，也就是说在一个函数体内不能包含另一个函数的定义，但是在调用一个函数的过程中却可以再调用另外一个函数，即C语言不能嵌套定义函数，但可以嵌套调用函数。

递归

如果定义一个概念需要用到这个概念本身，我们称它的定义是递归的。如果在调用一个函数的过程中又直接或间接地调用该函数本身，则称为函数的递归调用。

其实数学上有很多概念都用到递归了，比如n的阶乘（Factorial）是这样定义的：n的阶乘等于n乘以n-1的阶乘。那么n-1的阶乘是什么？是n-1乘以n-2的阶乘。那n-2的阶乘又是什么？这样下去永远也没完。因此需要定义一个最关键的基础条件：0的阶乘等于1。

0! = 1

n! = n * (n-1)!

因此，递归调用函数必须包含两部分：

①一般情况：由其自生定义的与原始问题类似的更小规模的子问题，它使递归过程持续进行

②基线情况：递归调用的最简形式，它是一个能够用来结束递归调用过程的条件

Example：Factorial阶乘问题

  • 一般情况：n! = n * (n-1)!
  • 基线情况：0! = 1

Example：Fibonacci数列问题

  • 一般情况：Fibonacci(n) = Fibonacci(n-1) + Fibonacci(n-2)
  • 基线情况：Fibonacci(0) = 0, Fibonacci(1) = 1

Example：Hanoi汉诺塔问题

问题描述：A,B,C表示三根柱子，A柱子从下往上放着按大小顺序的n片黄金圆盘，把这n片圆盘按大小顺序重新摆放到B柱子上，规定每次只能移动一个圆盘并且小圆盘上不能放大圆盘。

解题思路：(1)整体过程借助C将前n个圆盘从A移动到B;

(2)整体过程可分：

​ ①借助B将前n-1个圆盘从A移动到C;［借助移动］

​ ②将第n号圆盘从A直接移动到B; ［直接移动］

​ ③借助A将前n-1个圆盘从C移动到B;［借助移动］

  • 一般情况：n圆盘移动可分为n-1圆盘移动，依次下去
  • 基线情况：递归调用的最简形式是一个圆盘的移动情况

注意：

  • 每个迭代程序原则上都可以转换成等价的递归程序，反之不然，如Hanoi塔问题。
  • 从程序运行效率来看，递归函数在每次递归调用时都需要进行参数传递，现场保护等操作，增加了函数调用的时空开销，导致递归程序的时空效率偏低，因此为了提高程序的执行效率，应尽量用迭代形式替代递归形式。

函数栈帧

栈的特性：先进后出

栈帧：一个函数一个桟帧，变量的地址都存储在桟帧中

每个函数都有一个桟帧，但一个进程只有一个 data 段，当函数结束时，局部变量都会失效，桟帧也就消失了，但初始化的 static 变量是存储在 data 段的，所以 static 变量还是有效的。

局部变量与全局变量

局部变量与全局变量的定义

程序的编译单位是源程序文件，一个源文件可以包含一个或若干个函数；
在函数内部定义的变量是局部变量；而在函数外部定义的变量是外部变量(全局变量)；

(1)全局变量的特性

​ ①定义：不在任何语句块内定义的变量

​ ②作用域：整个程序

​ ③在整个程序运行中可以随时访问或修改全局变量

​ ④全局变量在不指定初值时会自动初始化为0

​ ⑤全局变量的适用情况：(满足以下三条即可将变量定义为全局变量)

​ a.变量的类型固定；

​ b.变量只有很有限几处需要修改,大部分为读取；

​ c.变量的值经常被程序中多个模块和函数使用；

​ ⑥全局变量破坏了函数的封装性，不建议使用

(3)局部变量的特性：在除整个程序以外的其他语句块内定义的变量

作用域和生命周期

变量的作用域规则：每个变量仅在定义它的语句块内有效，并且拥有自己的存储空间。

全局变量可为本文件中其他函数所共用，生命期为从变量的定义处开始到本程序文件的末尾。

在局部变量的作用范围内，局部变量有效，全局变量被屏蔽，即它不起作用。

变量的存储空间分配方式

(1)静态存储分配方式：在程序运行期间由系统分配固定的存储空间的方式，如全局变量

(2)动态存储分配方式：在程序运行期间根据需要动态的分配存储空间的方式，如自动变量

变量的存储类型

变量声明的一般形式：存储类型 数据类型 变量名表;

变量的存储类型是编译器为变量分配内存的方式，它决定变量的生命周期。C语言提供4种不同的存储类型：自动变量，寄存器变量，静态变量，外部变量。

(1)自动变量

​ ①定义格式：auto 类型名 变量名;(auto可以省略不写)

​ ②函数内定义变量如果没有指定存储类型，该变量的存储类型就缺省为auto；

​ 如果没有指定函数的返回值类型，那么函数的隐含类型为int型；

​ ③自动变量是局部变量，定义时必须初始化，退出函数时不会保持拥有值

​ ④在不同的并列语句块内可以定义同名变量，不会干扰，因为他们各自占据不同的内存单元，并且有着不同的 作用域

​ ⑤用const将形参声明为常量，可以有效防止形参值在函数内被修改

(2)寄存器变量

​ ①定义格式：register 类型名 变量名;

​ ②寄存器是CPU内部的一种容量有限但速度极快的存储器，寄存器变就是用寄存器存储的变量

​ ③使用情况：将使用频率较高的变量声明为register变量

(3)静态变量

​ ①定义格式：static 类型名 变量名;

​ ②静态变量的生存期是整个源程序，能够自动初始化为0，退出函数时保持拥有值

​ ③静态局部变量的作用域是限定在定义该变量的函数内；

​ 静态全局变量的作用域是限定在定义该变量的源文件内；

​ 非静态全局变量的生存期和作用域都是整个源程序；

(4)外部变量

​ ①定义格式：extern 类型名 变量名

​ ②函数外定义变量如果没有指定存储类型，那么该变量就是外部变量

​ ③外部变量是全局变量，自动初始化为0，生存期是整个源程序，作用域从定义点到文件结束

注意：

  • 按作用域的不同变量将分为2类：局部变量，全局变量；

  • 按存储类型不同变量将分为4类：自动变量，寄存器变量，静态变量，外部变量；

  • 自动变量就是局部变量，外部变量就是全局变量

  • 自动变量(局部变量)，不能自动初始化，定义时必须初始化，退出不保持拥有值；

  • 静态变量，自动初始化为0，只能初始化一次,退出保持拥有值；

  • 外部变量(全局变量)，自动初始化为0，在其他源文件内也可进行初始化；

    ​

一、局部变量的存储类型

(1) 自动变量：局部变量中，除 static 存储类别，都是自动变量，在每次调用函数时，都会重新给自动变量赋初值，如果不赋值，就会是随机的，内存采用动态存储分配方式，作用域是局部的。

(2) static 局部变量：在函数调用结束后，该变量占用的存储单元并不释放，保留当前值，下一次调用该函数时，该变量保持上次调用结束时的值，它只在编译时赋初值一次，如果在定义时没有赋值，将会采用默认值 0 和’\0’，采用静态存储分配方式，作用域也是局部的。

二、全局变量的存储类型

(1)扩展外部变量的作用域(extern)

extern 外部变量 —>表示把该外部变量的作用域扩展到此位置

如果一个程序包含两个.c 文件,两个文件都要用到同一个外部变量,不能分别定义两个同名的,否则会在程序连接时报“重复定义”的错误,这时,也只能用 extern 声明外部变量将该外部变量的作用域扩展到其他.c 文件中。

extern 既可以用来扩展外部变量在本文件中的作用域,也可以使外部变量的作用域从一个文件扩展到程序中的其他文件,在编译时当遇到 extern,系统会现在本文件中找外部变量的定义,如果找到就在本文件中扩展作用域,否则就在连接时从其他文件中找外部变量的定义,如果其他文件也找不到就报错

(2)限定外部变量的作用域(static)

static 外部变量 —>表示把该外部变量的作用域限制在本文件中

如果想把某些外部变量限定在本文件中使用,则可以在定义外部变量时加一个 static 声明，即静态外部变量,，这样声明以后，extern 对该变量就不再起作用，不会被其他文件所误用。

存储类型对局部变量和全局变量的不同意义

(1) 对局部变量，意义在于指定变量的存储方式和生存期的问题
(2) 对全局变量，都是编译时分配内存，都是静态存储分配方式，意义在于变量作用域的扩展问题

static声明变量的作用

(1) static 声明局部变量，使该变量从动态存储分配方式改为采用静态存储分配方式，生存期为该函数，但它所分配的内存空间在整个程序执行期间都始终存在。
(2) static 声名全局变量，该变量的作用域只限定在本文件内，但还是静态存储分配方式。

变量的声明和定义

变量的声明：不需要建立存储空间的声明，如 extern a变量的定义：建立存储空间的声明，如 int a

变量的声明包括定义，如 int a 既是声明也是定义，而 extern 是声明不是定义。外部变量的声明和定义是有区别的，定义只能有一次，位置在所有函数之外，而声明却可以有多次，且可在函数内也可在函数外，对外部变量的初始化只能在其定义时进行，声明的作用是说明该外部变量已在其他地方已定义，仅仅是为了扩展该外部变量的作用域。

内部函数和外部函数

内部和外部函数的定义

根据函数能否被其他源文件调用，将函数区分为“内部函数”和“外部函数”两种。

(1)内部函数：static 修饰的函数，只能被本文件中的其他函数调用，也称静态函数。

(2)外部函数：extern 修饰的函数，可供其他.c 文件调用，一般省略 extern，在需要调用此函数的其他文件中，需要对此函数作 extern 声明，即使在本函数中调用一个函数也要用函数原型进行声明。

扩展外部函数的作用域

(1)extern 声明外部函数，表示该外部函数在其他文件中已定义，将外部函数的作用域从其他文件扩展到该当前文件。
(2)函数原型声明，表示该函数在本文件中稍后定义，或在另一文件中定义将函数的作用域从定义处扩展到声明处(不必使用 extern)。

如#include 指令，头文件包含了许多函数原型，将头文件包含到该文件，相当于将头文件中的所有函数原型复制到该文件中，这样，这些函数的作用域就从定义处扩展到当前文件了。
