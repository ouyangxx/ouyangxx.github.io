Linux02-基础命令-目录操作与查找

目录操作

  • 根目录：linux系统下所有文件的路径都是以这个路径开始的，它是所有文件的顶级父目录
  • 绝对路径：以根(/)开头的路径。
  • 相对路径：不以根(/)开头的路径。

切换路径

$ cd /                     #切换到根目录

$ cd                     #切换到当前用户工作主目录

$ cd ~                     #切换到当前用户工作主目录

$ cd -                     #撤销上次的cd操作

$ cd .                     #一个点代表当前路径，切换到当前目录下等同于没切换

$ cd ..                     #两个点代表父目录，这里是切换到上级目录

$ cd ../..                     #切换到上两级目录

查看路径和文件

$ ls                     #显示当前工作路径下的文件

$ ls -l                     #显示当前工作路径下的文件，包括文件属性

$ ls -a                     #显示当前工作路径下的文件，包括隐藏文件

$ pwd                     #显示当前工作路径

$ tree                     #以树形分布显示当前工作路径下的所有文件

查找

Linux下有5个查找命令：grep，find，locate，whereis，which

grep查找

grep全称Global Regular Expression Print，它能使用正则表达式搜索文本，并把匹配的行打印出来。

命令格式：grep [option] pattern filename

filename：在这个文件里查找

pattern：搜索模式，可以是一个要匹配的字符串，或者是一个正则表达式

option：

$ grep cyril /etc/passwd             #在/etc/passwd这个文件中查找包含字符串”cyril”的行

$ grep “cyril” /etc/passwd             #在/etc/passwd这个文件中查找包含字符串”cyril”的行

$ grep ‘cyril’ /etc/passwd             #在/etc/passwd这个文件中查找包含字符串”cyril”的行

$ grep ‘test’ file*             #在当前目录下所有以file开始文件名的文件中查找包含字符串”test”的行

$ grep -i ‘#include’ main.c hello.c          #在main.c 和hello.c文件中忽略大小写查找包含字符串”#include”的行

$ grep ‘#include’ *             #在当前目录下所有文件中查找包含字符串”#include”的行

grep递归查找

$ grep -r ‘#include’ *             #在当前目录及子目录下所有文件中查找包含字符串”#include”的行

$ grep -l -r ‘#include’ *             #在当前目录及其子目录下查找包含字符串”#include”的行，但是不显示匹配的行，只显示匹配的文件

grep使用正则表达式进行匹配

$ ls -l | grep ‘^public’             #通过管道过滤ls -l输出的内容，只显示以字符串“public”开头的行

$ grep -n ‘goo*g’ readme.txt             #在readme.txt文件中查找一个字符串，这个字符串开头和结尾都是g，但是这两个g之间至少有一个o，像 gog,goog,goog这种，打印匹配的行，同时打印行号

其实，要用好grep这个工具，就是要写好正则表达式，这里不对grep的所有功能进行实例讲解，只列了几个例子
感受一下，以后会有专题讲解正则表达式的用法。

find查找

find是最常见和最强大的查找命令，你可以用它找到任何你想找的文件。

命令格式：find pathname -option [-print -exec -ok ...]

pathname：指定搜索目录，在这个目录里查找

[-print -exec -ok …]：指定对搜索结果的动作

-print：将匹配的文件输出到标准输出

-exec：对匹配的文件执行该参数所给出的shell命令，相应命令的形式为’command’ { } \;，注意{ }和\；之间的空格

-ok：和-exec的作用相同，只不过以一种更为安全的模式来执行该参数所给出的shell命令，在执行每一个命令之前，都会给出提示，让用户来确定是否执行

-option, 指定搜索的条件

注意以下几项option的区别：

-amin n         查找系统中最后N分钟访问的文件

-atime n         查找系统中最后n*24小时访问的文件

-cmin n         查找系统中最后N分钟被改变文件状态的文件

-ctime n         查找系统中最后n*24小时被改变文件状态的文件

-mmin n         查找系统中最后N分钟被改变文件数据的文件

-mtime n         查找系统中最后n*24小时被改变文件数据的文件

find执行shell命令

$ cat readme.txt             #打印readme.txt文件的内容

$ find readme.txt -ok cat {} \;             #打印readme.txt文件的内容

$ find readme.txt -exec cat {} \;             #打印readme.txt文件的内容

$ find readme.* | xargs cat             #打印所有以readme.开始文件名的文件的内容

之所以能用到xargs这个命令，是由于很多命令不支持|管道来传递参数，而日常工作中又有这个需求，所以就有
了xargs命令。xargs 可以读入 stdin 的数据，并且以空白字符或换行字符作为分隔符，将 stdin 的数据分隔成为
arguments。这里xargs读入了readme.*文件的内容而不是文件名，然后cat将文件内容给打印出来了。

按文件名查找

$ find ~ -name “*.log” -print             # 在当前用户home目录下查找所有以.log结束文件名的文件，并打印文件名

$ find . -name “*.log” -print             # 在当前目录下查找所有以.log结束文件名的文件，并打印文件名

$ find . -name “[A-Z]*” -print             # 在当前目录下查找所有以一个大写字母开始文件名的文件，并打印文件名

$ find /etc -name “host*” -print             # 在/etc目录下查找所有以host开始文件名的文件，并打印文件名

$ find . -name “[a-z]*[0-9].log” -print  # 在当前目录下查找所有以一个小写字母开始并且以一个数字和.log结束文件名的文件，并打印文件名

按文件权限查找

find . -perm 0644 -print             #在当前目录下查找所有文件权限为0777的文件

按文件拥有者查找

$ find /etc/ -user cyril -print             #在/etc/目录下查找所有文件属主为cyril的文件

按文件更改时间和访问时间查找

$ find /home/cyril -mtime -5 -print             #在/home/finance/目录下查找所有更改时间在5日以内的文件

$ find . -newer h1.txt ! -newer h5.txt           #在当前目录下查找所有比h1.txt文件新但比h5.txt旧的文件

按文件类型查找

$ find . -type f             #在当前目录下查找所有的普通文件

$ find . -type d             #在当前目录下查找所有的文件夹

$ find . ! -type d             #在当前目录下查找除文件夹外的所有文件

按文件大小查找

$ find . -size +10 -print             #在当前目录下查找所有文件大小超过10块(1块＝512字节)的文件

$ find . -size +1000000c -print             #在当前目录下查找所有文件大小超过1M字节的文件

$ find . -size 1000000c -print             #在当前目录下查找所有文件大小刚好为1M字节的文件

注意：find命令本身就是递归查找的，当前目录还未查找完时若遇到一个子目录，则默认先在子目录中查找，子目
录查完了再继续当前目录的查找。

若想强制在进入子目录前先行查找完本目录，可以使用-depth选项。

$ find / -name “*.config” -depth -print             #从根目录开始一级一级的查找所有以.config结束文件名的文件

locate查找

locate命令其实是“find -name”的另一种写法，但是要比后者快得多，原因在于它不搜索具体目录，而是搜索一个
数据库（/var/lib/locatedb），这个数据库中含有本地所有文件信息。Linux系统自动创建这个数据库，并且每天
自动更新一次，所以使用locate命令查不到最新变动过的文件。为了避免这种情况，可以在使用locate之前，先使
用updatedb命令，手动更新数据库，当然执行这个更新命令需要有管理员权限。

$ locate -i ~/h             #查找home目录下所有以h开始文件名的文件，忽略大小写

$ locate /etc/sh             #查找/etc目录下所有以sh开始文件名的文件

whereis查找

whereis命令只能用于程序名的搜索，而且只搜索二进制文件（参数-b）、man说明文件（参数-m）和源代码文
件（参数-s）。如果省略参数，则返回所有信息。

$ whereis grep             #查找grep程序的位置

which查找

which命令的作用是，在PATH变量指定的路径中，搜索某个系统命令的位置，并且返回第一个搜索结果。也就是
说，使用which命令，就可以看到某个系统命令是否存在，以及执行的到底是哪一个位置的命令。

$ which grep             #查看grep命令是否存在
